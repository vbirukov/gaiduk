# Иконки и Motion для аудиоплеера Гайдука

Версия: 1.0  
Автор: Vitaliy + AI‑art director

---

## 1. Общие принципы

- Векторные, плоские, без фотореализма – ближе к комикс/плакату.
- Скруглённые углы, живые контуры (не абсолютно геометрические, допускается лёгкий «ручной» вайб).
- Простые силуэты, читаемость на размерах 24–32 px.
- Иконки должны одинаково хорошо смотреться и в тёмной «Rasta Night», и в светлых темах («Rasta Light», «Jaipur»).

---

## 2. Сетка и размеры иконок

```text
Основная иконка UI:     24 × 24 px (controls, toolbar)
Крупные action‑иконки:  32 × 32 px (Play, Pause, Next, Previous)
Сервисные иконки:       20 × 20 px (настройки, инфо, share)

Сетка: 24px с отступами 2px по периметру (эффективная рабочая зона – 20×20px).
Толщина линий: 2px (1.5px допустимо для деталей внутри 32px иконок).
Радиусы скругления: 2–3px для углов, 12–14px для круговых иконок.
```

Стилизация: форма может быть слегка асимметричной (hand‑drawn), но bounding box и оптический вес должны совпадать.

---

## 3. Пиктограммы и метафоры

### 3.1. Базовый набор

- Play / Pause / Next / Previous
- Favorite / Like (сердечко или листочек)
- Download / Offline
- Playlist / Queue
- Search / Filter
- Settings
- Profile / User
- Info / About

### 3.2. Специальные растаманские и сказочные

- **Leaf** – иконка раздела «Растаманские сказки» (упрощённый лист конопли, без гиперреализма).
- **Sun / Mandala** – сказочный портал, может использоваться как иконка главной или «Все сказки».
- **Bong / Pipe / Cloud** – аккуратная, стилизованная иконка «дымка» или «атмосфера».
- **Drum / Djembe** – раздел с джемами и живыми выступлениями.
- **Tape / Cassette / Reel** – офлайн записи, архив.

Важно: специальные иконки не перегружаем деталями – максимум 2–3 ключевые линии внутри.

---

## 4. Цвет и стилизация по темам

### 4.1. Rasta Night (тёмная)

- Контур иконок: `Text High` (смотри цвет в night‑стайлгайде).
- Фон action‑иконок: круг/кнопка `Rasta Sun`.
- Hover: лёгкое светящееся кольцо (glow‑primary).

### 4.2. Rasta Light (яркая)

- Контур: `Ink Line` (#222222).
- Заливки: `Rasta Green` / `Rasta Yellow` / `Psycho Orange`.
- Hover: изменение фона (жёлтый/зелёный) + увеличенный scale 1.04.

### 4.3. Jaipur

- Контур: `Havelis White` на терракотовом фоне или `Text Palace Dark` на светлом.
- Фон: мягкий `Jaipur Facade` / `Jaipur Blush` с белой обводкой.

Рекомендуется использовать один SVG‑набор и перекрашивать через CSS/`currentColor`.

- Исходник: Figma/Sketch с сеткой 24×24 и 32×32.
- Экспорт: SVG с `stroke-linecap="round"` и `stroke-linejoin="round"`.
- Цвет линий – `currentColor`, чтобы иконки подкрашивались темой.
- Файловая структура (пример):

## 6. Motion: принципы

Общая идея motion – живость, но без навязчивости. Всё должно «дышать» лёгким регги‑ритмом, не мешая слушать сказки.

- **Длительность:** короткие действия 150–220 ms, большие переходы 250–350 ms.
- **Кривая:** `cubic-bezier(0.17, 0.67, 0.19, 1.24)` – лёгкий overshoot, мягкий «пружинный» эффект.
- **Частота:** не анимировать всё одновременно; не более 1–2 акцентных анимаций на экране.

---

## 7. Motion‑паттерны для иконок

### 7.1. Hover на контролах плеера

- Лёгкий масштаб: `scale(1.05)`.
- Сдвиг по оси Y: `translateY(-1px)`.
- Небольшой glow для основных кнопок.

CSS‑пример:

```css
.icon-button {
  transition:
    transform 180ms cubic-bezier(0.17, 0.67, 0.19, 1.24),
    box-shadow 180ms cubic-bezier(0.17, 0.67, 0.19, 1.24);
}

.icon-button:hover {
  transform: translateY(-1px) scale(1.05);
  box-shadow: 0 0 16px rgba(249, 199, 79, 0.6); /* для Rasta Light */
}
```

### 7.2. Нажатие (tap/click)

- Быстрый отклик: scale до `0.96` и лёгкий затемнённый фон.
- Возврат к исходному состоянию с коротким overshoot.

### 7.3. Состояние Play ↔ Pause

- Анимация смены иконки через morph (если используете SVG‑анимацию/Framer Motion) или через кросс‑fade.
- Длительность: 180–220 ms.
- С лёгким вращением диска/кассеты в фоне (например, `rotate(5deg)` туда‑сюда).

---

## 8. Motion для экранов

### 8.1. Переходы между экранами

- Горизонтальный slide: каталог → плеер → настройки.
- Длительность: 250–300 ms.
- Кривая: та же лёгкая пружина.

Пример с Framer Motion:

```tsx
<motion.div
  initial={{ x: 40, opacity: 0 }}
  animate={{ x: 0, opacity: 1 }}
  exit={{ x: -40, opacity: 0 }}
  transition={{ duration: 0.26, ease: [0.17, 0.67, 0.19, 1.24] }}
>
  {children}
</motion.div>
```

### 8.2. Появление мини‑плеера

- Выезд снизу на 12–16 px.
- Одновременное лёгкое масштабирование контента (`scale(1.02)` → `1`).
- Появление за 220 ms.

### 8.3. Анимация прогресс‑бара

- Линейное движение, без резких рывков.
- В активной теме можно добавить едва заметное колебание яркости градиента (например, через CSS‑анимацию opacity у overlay).

---

## 9. Специальные анимации для раста‑темы

Использовать очень дозировано (например, только в сплэше или при достижении достижений).

- **Дыхание солнца/мандалы:** медленное (6–8 секунд) пульсирование масштаба `1 → 1.03 → 1` и лёгкое вращение `-2deg → 2deg`.
- **Дым‑волна:** SVG‑shape облака/дыма, который медленно плывёт (10–15 секунд) и слегка меняет форму/прозрачность.
- **Прыгающие ноты:** при смене трека маленькие ноты выскакивают из иконки play и падают вниз, исчезая.

Важно: отключать или сильно уменьшать эти анимации в настройках для пользователей, которые не любят лишний motion.

## 10. Интеграция в код

Предлагаемая структура (можно адаптировать под текущий проект):

```text
src/
  components/
    IconButton/
      IconButton.tsx       – общий контейнер с hover/press анимацией
    PlayerControls/
      PlayerControls.tsx   – play/pause/next/prev, использует IconButton
  icons/                   – набор SVG‑иконок
  theme/
    rastaNightTheme.ts
    rastaLightTheme.ts
    jaipurTheme.ts
    motion.ts              – константы easing, durations
```

`motion.ts` может экспортировать:

```ts
export const motion = {
  fast: 0.18,
  normal: 0.26,
  slow: 0.34,
  easeSpring: [0.17, 0.67, 0.19, 1.24],
};
```

А затем использоваться везде, где есть анимации (Framer Motion, CSS‑переменные и т.д.).

## 11. Чек‑лист по иконкам и motion

1. Сделать сетку 24/32px и нарисовать базовый набор иконок в едином стиле.
2. Экспортировать как SVG с `currentColor` и подключить как React‑компоненты.
3. Вынести общие параметры анимаций (длительность, easing) в отдельный модуль.
4. Реализовать IconButton с единым поведением hover/press для всех контролов.
5. Настроить экранные переходы (catalog → player → settings) с мягкими slide‑анимациями.
6. Добавить одну‑две фирменные спец‑анимации (солнце/мандала, дым), но убедиться, что они не мешают UX.
